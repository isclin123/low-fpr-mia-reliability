## What Belongs in the Public Deposit

Include:

- project-authored code and examples;
- pinned environment lock;
- reproduction commands and manifests;
- processed/split arrays where upstream terms permit redistribution;
- score-array examples;
- final manuscript tables and appendix tables;
- generated figure source data and final figures;
- reproducibility QA records;
- Data/Code Availability statement;
- release route/licence notes.

Do not include by default:

- raw third-party datasets if upstream redistribution terms are unclear;
- the entire `analysis/results/` directory as Git content;
- local caches or temporary virtual environments;
- private process notes that are not needed for public reproducibility.

Large generated results:

- host in Zenodo as separate archival files if needed;
- otherwise provide regeneration commands, expected output counts and checksums/manifests.
